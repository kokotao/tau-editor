# 跨平台文本编辑器 - 需求文档

**版本：** 1.0  
**创建日期：** 2026-03-11  
**状态：** 草案

---

## 1. 目标用户群体分析

### 1.1 核心用户画像

| 用户类型 | 描述 | 占比预估 |
|---------|------|---------|
| 开发者/程序员 | 日常编写代码、查看日志、编辑配置文件 | 45% |
| 技术写作者 | 编写文档、技术博客、API 文档 | 20% |
| 数据分析师 | 处理 CSV、JSON、XML 等数据文件 | 15% |
| 学生/教育工作者 | 学习笔记、作业、研究文档 | 12% |
| 普通办公用户 | 日常文本编辑、简单笔记 | 8% |

### 1.2 用户特征

- **技术水平：** 中等到高级，熟悉基本命令行操作
- **使用频率：** 每日使用，单次使用时长 30 分钟 -4 小时
- **设备环境：** 多设备切换（桌面 + 笔记本 + 平板）
- **痛点：** 现有编辑器要么太重（IDE），要么功能太弱（记事本）

### 1.3 使用场景

| 场景 | 描述 | 频率 |
|-----|------|-----|
| 快速查看日志文件 | 打开大型日志文件，搜索关键词 | 高频 |
| 编辑配置文件 | 修改 YAML/JSON/XML 配置 | 高频 |
| 编写 Markdown 文档 | 技术文档、笔记、README | 高频 |
| 代码片段编辑 | 临时编写/修改代码片段 | 中频 |
| 数据文件处理 | 查看/编辑 CSV、JSON 数据 | 中频 |
| 批量文本替换 | 多文件查找替换 | 低频 |

## 3. MVP（最小可行产品）功能范围

### 3.1 MVP 目标
**在 8 周内交付一个可用的跨平台文本编辑器，满足 80% 核心用户的核心需求。**

### 3.2 MVP 包含功能

| 模块 | 功能 | 优先级 |
|-----|------|-------|
| 文件操作 | 新建、打开、保存、另存为 | P0 |
| 基础编辑 | 撤销/重做、剪切复制粘贴 | P0 |
| 搜索替换 | 当前文件查找替换（含正则） | P0 |
| 语法高亮 | 支持目标格式的高亮 | P0 |
| 多标签 | 多标签页切换 | P0 |
| 编码支持 | UTF-8 自动检测与转换 | P0 |
| 主题 | 浅色/深色主题切换 | P1 |
| 自动保存 | 可配置间隔自动保存 | P1 |
| 行号显示 | 行号开关 | P1 |
| Markdown 预览 | 实时预览 | P1 |
| JSON 格式化 | 格式化/压缩 | P1 |

### 3.3 MVP 不包含功能（后续迭代）

| 功能 | 说明 | 计划版本 |
|-----|------|---------|
| 插件系统 | 扩展生态 | v2.0 |
| 云端同步 | 跨设备同步 | v2.0 |

## 4. 用户故事与验收标准

### 4.1 文件管理

#### US-001: 创建新文件
**作为** 用户  
**我想要** 快速创建新文件  
**以便于** 开始编写内容  

**验收标准：**
- [ ] 通过菜单"文件 > 新建"创建新文件
- [ ] 通过快捷键 Ctrl/Cmd+N 创建新文件
- [ ] 新文件默认命名为"未命名 -1"，后续递增
- [ ] 新文件自动打开并聚焦到编辑区
- [ ] 未保存的新文件关闭时提示保存

#### US-002: 打开现有文件
**作为** 用户  
**我想要** 打开已有的文件  
**以便于** 继续编辑或查看内容  

**验收标准：**
- [ ] 通过菜单"文件 > 打开"选择文件
- [ ] 通过快捷键 Ctrl/Cmd+O 打开文件选择器
- [ ] 支持拖拽文件到编辑器窗口打开
- [ ] 支持从命令行参数打开文件
- [ ] 打开大文件（>100MB）时显示加载进度
- [ ] 不支持的格式给出友好提示

#### US-003: 保存文件
**作为** 用户  
**我想要** 保存我的编辑内容  
**以便于** 不丢失工作成果  

**验收标准：**
- [ ] 通过菜单"文件 > 保存"保存当前文件
- [ ] 通过快捷键 Ctrl/Cmd+S 保存
- [ ] 新文件保存时弹出保存对话框
- [ ] 已有文件保存时直接覆盖
- [ ] 保存失败时显示错误原因（权限、磁盘满等）
- [ ] 保存成功后状态栏显示"已保存"提示

#### US-004: 自动保存
**作为** 容易忘记保存的用户  
**我想要** 编辑器自动保存我的更改  
**以便于** 意外关闭时不丢失内容  

**验收标准：**
- [ ] 可在设置中开启/关闭自动保存
- [ ] 可配置自动保存间隔（1/5/10/30 分钟）
- [ ] 自动保存时不中断用户输入
- [ ] 自动保存的文件有明确标识
- [ ] 崩溃后可恢复自动保存的内容

### 4.2 编辑功能

#### US-005: 撤销与重做
**作为** 用户  
**我想要** 撤销我的错误操作  
**以便于** 恢复到之前的状态  

**验收标准：**
- [ ] 通过菜单"编辑 > 撤销"执行撤销
- [ ] 通过快捷键 Ctrl/Cmd+Z 撤销
- [ ] 通过快捷键 Ctrl/Cmd+Shift+Z 或 Ctrl/Cmd+Y 重做
- [ ] 撤销/重做历史无限制（受内存限制）
- [ ] 撤销栈在保存后不清空

#### US-006: 查找与替换
**作为** 用户  
**我想要** 在文件中查找和替换文本  
**以便于** 快速定位和批量修改内容  

**验收标准：**
- [ ] 通过快捷键 Ctrl/Cmd+F 打开查找框
- [ ] 通过快捷键 Ctrl/Cmd+H 打开替换框
- [ ] 支持区分大小写选项
- [ ] 支持正则表达式搜索
- [ ] 支持全文件替换和逐个确认替换
- [ ] 查找时高亮所有匹配项
- [ ] 显示匹配结果数量

#### US-007: 多标签编辑
**作为** 需要同时编辑多个文件的用户  
**我想要** 在同一个窗口中打开多个标签页  
**以便于** 快速切换不同文件  

**验收标准：**
- [ ] 支持同时打开至少 20 个标签页
- [ ] 标签页显示文件名和关闭按钮
- [ ] 点击标签页切换文件
- [ ] 支持拖拽标签页重新排序
- [ ] 支持 Ctrl/Cmd+W 关闭当前标签
- [ ] 支持 Ctrl/Cmd+Tab 切换标签
- [ ] 未保存的标签页关闭时提示保存

### 4.3 语法高亮与预览

#### US-008: 语法高亮
**作为** 开发者  
**我想要** 代码和标记语言有语法高亮  
**以便于** 更容易阅读和理解内容  

**验收标准：**
- [ ] 根据文件扩展名自动识别语言
- [ ] 支持目标格式（txt, md, json, xml, yaml, csv, html, 常见代码语言）
- [ ] 高亮颜色符合主题配色
- [ ] 支持手动选择语言（当自动识别失败时）
- [ ] 高亮性能良好（10000 行文件无明显卡顿）

#### US-009: Markdown 实时预览
**作为** 技术写作者  
**我想要** 边编辑边预览 Markdown 效果  
**以便于** 确认最终渲染效果  

**验收标准：**
- [ ] 通过菜单或快捷键切换预览模式
- [ ] 支持分屏预览（左编辑右预览）
- [ ] 预览实时更新（输入后<500ms）
- [ ] 支持渲染表格、代码块、数学公式
- [ ] 支持预览中的链接点击跳转

#### US-010: JSON 格式化
**作为** 开发者  
**我想要** 一键格式化 JSON 文件  
**以便于** 阅读和调试 JSON 数据  

**验收标准：**
- [ ] 通过菜单"格式 > 格式化 JSON"执行
- [ ] 通过快捷键执行格式化
- [ ] 支持格式化（美化）和压缩（单行）两种模式
- [ ] 格式错误时显示错误位置和原因
- [ ] 格式化前自动备份原内容（可撤销）

### 4.4 跨平台体验

#### US-011: 一致的用户体验
**作为** 多平台用户  
**我想要** 在不同操作系统上有一致的体验  
**以便于** 无需重新学习  

**验收标准：**
- [ ] UI 布局在各平台保持一致
- [ ] 快捷键遵循各平台惯例（Ctrl vs Cmd）
- [ ] 文件对话框使用原生组件
- [ ] 字体渲染在各平台清晰可读
- [ ] 深色模式在各平台正常工作

#### US-012: 性能要求
**作为** 用户  
**我想要** 编辑器响应迅速  
**以便于** 流畅地工作  

**验收标准：**
- [ ] 启动时间 < 2 秒（冷启动）
- [ ] 打开 10MB 文件时间 < 3 秒
- [ ] 输入延迟 < 50ms
- [ ] 搜索 10000 行文件时间 < 1 秒
- [ ] 内存占用 < 500MB（正常编辑场景）

## 5. 非功能性需求

### 5.1 性能要求

| 指标 | 目标值 | 测量方法 |
|-----|-------|---------|
| 启动时间 | < 2 秒 | 点击图标到可编辑状态 |
| 文件打开 | < 3 秒（10MB） | 选择文件到内容显示 |
| 输入延迟 | < 50ms | 按键到字符显示 |
| 搜索速度 | < 1 秒（10000 行） | 输入搜索词到结果高亮 |
| 内存占用 | < 500MB | 正常编辑 3-5 个文件 |

### 5.2 兼容性要求

- **Windows:** Windows 10 (1903+) / Windows 11, 64 位
- **macOS:** macOS 11 (Big Sur) 及以上，支持 Intel 和 Apple Silicon
- **Linux:** Ubuntu 20.04+, Debian 11+, Fedora 35+，glibc 2.31+

### 5.3 安全要求

- [ ] 不自动上传用户文件到云端
- [ ] 本地存储的配置文件加密敏感信息
- [ ] 打开文件前进行基本的安全检查
- [ ] 不执行文件中的任何代码

### 5.4 可访问性要求

- [ ] 支持系统字体缩放
- [ ] 键盘可完成所有操作
- [ ] 支持屏幕阅读器（基础）
- [ ] 颜色对比度符合 WCAG 2.1 AA 标准

## 7. 风险与假设

### 7.1 关键假设

| 假设 | 验证方式 | 风险等级 |
|-----|---------|---------|
| 用户需要轻量级编辑器而非 IDE | 用户访谈 | 中 |
| 跨平台需求真实存在 | 市场调研 | 低 |
| 用户愿意尝试新编辑器 | 早期用户测试 | 中 |
| 开源竞品不会快速跟进 | 竞争分析 | 低 |

### 7.2 主要风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|------|------|---------|
| 性能不达标 | 高 | 中 | 早期性能测试，优化核心模块 |
| 跨平台兼容问题 | 高 | 中 | 持续集成测试，多平台测试机 |
| 用户获取困难 | 中 | 中 | 社区运营，开源部分组件 |
| 范围蔓延 | 高 | 高 | 严格 MVP 范围控制 |
