# 客户端体验改造实施清单（2026-03-14）

## 1. 目标与范围

本次改造覆盖以下 5 个需求：

1. 解决客户端右上角关闭按钮（叉叉）无法关闭应用的问题。
2. 为 `.md` 文档增加实时预览能力，并支持 Mermaid 流程图渲染。
3. 去掉启动时提示面板与“打开成功”类通知（例如“文件已打开/工作区已打开/编辑器已就绪”）。
4. 左侧面板支持自由开关，并增加分栏快捷图标按钮（含快捷键和可见手柄）。
5. 左侧资源管理器目录结构交互优化，符合主流编辑器设计（树节点可折叠/展开，视觉与交互一致）。

不在本期范围：

1. 多窗口协同编辑。
2. Markdown 导出 PDF/HTML。
3. 自定义 Mermaid 主题配置面板。

## 3. 改造分解（按优先级）

## A. P0 关闭按钮不可关闭（稳定性阻断）

### A1. 行为约束

1. 无未保存内容：点击关闭直接退出。
2. 有未保存内容：仅弹一次确认；确认后必须成功退出。
3. 确认取消：保持应用打开，不丢失状态。

### A2. 代码任务

1. `frontend/src/services/windowService.ts`
   - 增加“关闭请求状态锁”，防止 `onCloseRequested` 重入。
   - 保留 `forceClosing`，并将 `detach()` 调用前移到最终关闭路径。
   - 新增兜底关闭策略（`currentWindow.close()` 失败时 fallback）。
2. `frontend/src/App.vue`
   - 保证卸载时 `windowService.detach()` 仅执行一次，不与主动 close 冲突。

### A3. 测试任务

1. `tests/unit/services/windowService.test.ts`
   - 增加 dirty/non-dirty + confirm true/false 分支。
   - 增加“确认后不再阻塞关闭”断言。
2. `tests/e2e/specs` 新增 `window-close.spec.ts`
   - 覆盖实际点击关闭按钮行为（至少 1 条 happy path）。

### A4. 验收标准

1. 手工测试 20 次连续关闭无卡死。
2. 有未保存内容时确认后退出成功率 100%。

## B. P0 Markdown 实时预览 + Mermaid

### B1. 技术选型

建议引入依赖（在 `frontend/package.json`）：

1. `marked`（Markdown 解析）
2. `dompurify`（HTML 清洗）
3. `mermaid`（流程图渲染）

### B2. 交互设计

1. 仅在当前标签语言为 `markdown` 时显示预览能力。
2. 提供 3 种模式：
   - `edit` 仅编辑
   - `split` 分栏（默认）
   - `preview` 仅预览
3. 输入后防抖刷新（120~200ms）。

### B3. 代码任务

1. `frontend/src/services/markdownService.ts`（新增）
   - `renderMarkdown(raw: string): string`
   - `renderMermaid(containerEl): Promise<void>`
2. `frontend/src/components/editor/MarkdownPreview.vue`（新增）
   - 接收 `content/theme`，渲染清洗后的 HTML。
   - 识别 `pre code.language-mermaid` 并调用 Mermaid。
3. `frontend/src/App.vue`
   - 在 `activeTab.language === 'markdown'` 时，替换原单编辑区为分栏容器。
   - 将模式状态持久化（可放入 settings store）。
4. `frontend/src/components/editor/Toolbar.vue`
   - 增加预览模式切换图标（仅 markdown 时可用）。

### B4. 测试任务

1. `tests/unit/components/MarkdownPreview.test.ts`（新增）
   - 渲染基础 markdown、代码块、mermaid 块、错误 mermaid。
2. `tests/unit/services/markdownService.test.ts`（新增）
   - XSS 清洗断言。
3. `tests/e2e/specs/markdown-preview.spec.ts`（新增）
   - 输入 markdown 后实时看到预览内容。
   - mermaid 图可见。

### B5. 验收标准

1. `.md` 文件输入后 300ms 内可见预览变化。
2. Mermaid 语法错误时不影响编辑器可用性。
3. 普通代码文件编辑体验不退化。

## E. P1 资源管理器目录结构优化（主流设计对齐）

### E1. 设计目标（对齐主流编辑器）

1. 文件夹节点支持清晰的折叠/展开状态（箭头方向、动画、层级缩进一致）。
2. 文件与文件夹图标语义统一（文件夹闭合/展开两态，文件按类型图标）。
3. 选中态、悬浮态、焦点态一致，且与当前主题（深色/浅色）对比充分。
4. 支持常见交互习惯：
   - 单击选中
   - 双击或 Enter 打开文件
   - 左箭头折叠、右箭头展开（焦点在目录树时）
5. 面板折叠后恢复时保留上次展开状态（不丢失用户上下文）。

### E2. 代码任务

1. `frontend/src/components/editor/FileTree.vue`
   - 优化节点结构样式：统一缩进规则、节点行高、图标对齐、hover/active/focus 态。
   - 规范 folder toggle 区域点击热区，避免误触。
   - 增加键盘导航支持（ArrowUp/Down/Left/Right、Enter）。
   - 为树节点增加可测的 `data-testid`（folder/file/toggle/state）。
2. `frontend/src/stores/fileSystem.ts`
   - 补充目录展开状态持久化策略（按路径记录 expanded map）。
   - 与工作区刷新逻辑对齐：刷新后尽量保留已展开节点。
3. `frontend/src/App.vue`
   - 面板收起/展开时不重置 file tree 组件关键状态。
4. `frontend/src/components/editor/Toolbar.vue`
   - 面板折叠按钮增加 tooltip 文案区分：
     - `折叠资源管理器`
     - `展开资源管理器`

### E3. 测试任务

1. `tests/unit/components/FileTree.test.ts`
   - 新增：文件夹折叠/展开图标状态切换。
   - 新增：键盘左右箭头触发展开/折叠。
   - 新增：面板隐藏再显示后展开状态保持。
2. `tests/e2e/specs/sidebar-drawer.spec.ts`
   - 补充目录树交互流：展开多级目录 -> 折叠面板 -> 展开面板 -> 状态仍在。

### E4. 验收标准

1. 任意目录节点都可稳定折叠/展开，无“点不中”或误触。
2. 左侧面板折叠后再展开，目录展开状态保持。
3. 目录树可通过键盘完成基本导航与折叠/展开。
4. 视觉样式符合主流 IDE/编辑器习惯（层级清晰、图标语义明确、选中高亮稳定）。

## 6. 风险与回滚

主要风险：

1. Mermaid 渲染导致预览性能抖动。
2. 关闭逻辑在不同平台存在行为差异。
3. 布局改动影响现有 e2e 选择器稳定性。

回滚策略：

1. 通过独立提交拆分，优先回滚 Markdown 预览提交，不回滚关闭修复。
2. 对新功能加开关（如 `settings.markdownPreviewEnabled`），必要时线上快速关闭。
