## US-FILE-001: 创建新文件

### 故事描述
**作为** 用户  
**我想要** 快速创建新文件  
**以便于** 立即开始编写内容，无需先选择保存位置

### 验收标准
```
Given 编辑器已打开
When 用户点击菜单"文件 > 新建"或按下 Ctrl/Cmd+N
Then 创建一个名为"未命名 -1"的新标签页
And 编辑区为空且可输入
And 光标自动聚焦到编辑区
```

```
Given 已存在"未命名 -1"文件
When 用户再次创建新文件
Then 新文件命名为"未命名 -2"，后续递增
```

```
Given 存在未保存的新文件
When 用户尝试关闭该标签页
Then 弹出对话框询问"是否保存更改？"
And 提供"保存"、"不保存"、"取消"三个选项
```

### 技术备注
- 新文件默认编码为 UTF-8
- 新文件默认语言模式为纯文本
- 标签页标题显示为斜体表示未保存状态

## US-FILE-002: 打开现有文件

### 故事描述
**作为** 用户  
**我想要** 打开已有的文件进行编辑  
**以便于** 继续之前的工作或查看文件内容

### 验收标准
```
Given 编辑器已打开
When 用户点击菜单"文件 > 打开"或按下 Ctrl/Cmd+O
Then 弹出系统文件选择对话框
And 用户可选择任意文本文件
And 选中后文件在新标签页中打开
```

```
Given 用户拖拽文件到编辑器窗口
When 文件被释放
Then 文件自动在新标签页中打开
And 显示加载进度（如文件>10MB）
```

```
Given 用户从命令行启动编辑器并传入文件路径
When 编辑器启动
Then 指定的文件自动打开
```

```
Given 用户尝试打开不支持的文件格式（如二进制文件）
When 文件被检测为不支持格式
Then 显示友好提示"此文件格式不支持，请选择文本文件"
And 文件不打开
```

```
Given 用户尝试打开超过 500MB 的文件
When 文件被检测为超大文件
Then 显示警告"文件较大，可能导致性能问题，是否继续打开？"
And 用户确认后继续打开
```

### 技术备注
- 支持的文件格式见《MVP 功能需求文档》3.4 节
- 大文件加载需使用流式读取，避免内存溢出
- 文件编码自动检测，检测失败时提示用户手动选择

## US-FILE-003: 保存文件

### 故事描述
**作为** 用户  
**我想要** 保存我的编辑内容  
**以便于** 确保工作成果不丢失

### 验收标准
```
Given 用户正在编辑一个新文件（未命名）
When 用户点击"文件 > 保存"或按下 Ctrl/Cmd+S
Then 弹出系统保存对话框
And 用户可选择保存位置和文件名
And 确认后文件保存到指定位置
And 标签页标题更新为实际文件名
And 状态栏显示"已保存"提示（2 秒后消失）
```

```
Given 用户正在编辑一个已有文件
When 用户点击"文件 > 保存"或按下 Ctrl/Cmd+S
Then 文件直接保存到原位置（覆盖）
And 标签页的"未保存"标识消失
And 状态栏显示"已保存"提示（2 秒后消失）
```

```
Given 保存操作失败（如权限不足、磁盘满）
When 保存尝试执行
Then 显示错误对话框，说明失败原因
And 提供"重试"、"另存为"、"取消"选项
And 文件保持未保存状态
```

```
Given 文件在外部被修改
When 用户尝试保存
Then 提示"文件在外部被修改，是否覆盖？"
And 提供"覆盖"、"重新加载"、"取消"选项
```

### 技术备注
- 保存时先写入临时文件，成功后再替换原文件（原子操作）
- 保存前检查磁盘空间
- 保存操作不应阻塞 UI，大文件保存时显示进度

## US-FILE-004: 另存为

### 故事描述
**作为** 用户  
**我想要** 将当前文件另存为新文件  
**以便于** 保留原文件的同时创建副本或更改格式

### 验收标准
```
Given 用户正在编辑任意文件
When 用户点击"文件 > 另存为"或按下 Ctrl/Cmd+Shift+S
Then 弹出系统保存对话框
And 默认文件名与当前文件相同
And 用户可更改文件名和位置
And 确认后保存为新文件
And 编辑器切换到新文件标签页
And 原文件保持打开状态（未修改）
```

```
Given 用户另存为时选择了已存在的文件名
When 用户确认保存
Then 弹出对话框询问"文件已存在，是否覆盖？"
And 用户确认后才覆盖
```

```
Given 用户另存为时选择了不同的文件扩展名
When 保存完成
Then 文件内容保持不变
And 语法高亮根据新扩展名自动调整
```

### 技术备注
- 另存为后，新文件成为当前编辑文件
- 原文件标签页保持打开但标记为"已保存"状态
- 编码格式保持与原文件一致，除非用户手动更改

## US-FILE-005: 拖拽打开文件

### 故事描述
**作为** 用户  
**我想要** 通过拖拽快速打开文件  
**以便于** 减少操作步骤，提高效率

### 验收标准
```
Given 编辑器窗口已打开
When 用户从文件管理器拖拽一个文件到编辑器窗口
Then 文件被高亮接受区域（窗口边框变色）
And 释放鼠标后文件在新标签页中打开
```

```
Given 用户同时拖拽多个文件（<10 个）
When 释放鼠标
Then 所有文件依次在新标签页中打开
And 最后一个文件为活动标签页
```

```
Given 用户拖拽不支持的文件格式
When 释放鼠标
Then 显示提示"不支持的文件格式"
And 不打开任何文件
```

### 技术备注
- 拖拽区域为整个编辑器窗口
- 多文件拖拽时按顺序打开，避免同时加载导致卡顿
- 拖拽文件夹时，提示"请选择具体文件"

## US-FILE-006: 自动保存

### 故事描述
**作为** 容易忘记保存的用户  
**我想要** 编辑器自动保存我的更改  
**以便于** 意外关闭或崩溃时不丢失内容

### 验收标准
```
Given 用户已开启自动保存功能
When 用户编辑文件且超过设定时间未手动保存
Then 编辑器自动保存当前内容
And 状态栏显示"自动保存于 HH:MM"
And 不影响用户继续输入
```

```
Given 用户在设置中配置自动保存
When 用户选择自动保存间隔
Then 可选值为：1 分钟、5 分钟、10 分钟、30 分钟、关闭
And 设置立即生效
```

```
Given 自动保存触发时文件从未保存过（新文件）
When 自动保存执行
Then 内容保存到临时目录
And 标签页仍显示"未保存"状态
And 崩溃恢复时可从临时目录恢复
```

```
Given 自动保存失败（如磁盘满）
When 保存尝试执行
Then 状态栏显示警告"自动保存失败"
And 不中断用户编辑
And 5 分钟后重试
```

### 技术备注
- 自动保存不改变文件的"未保存"标识，只有手动保存才清除
- 临时保存文件存储在系统临时目录，带时间戳
- 自动保存时不触发"文件外部修改"检测

## US-FILE-007: 关闭文件

### 故事描述
**作为** 用户  
**我想要** 关闭不再需要的文件标签页  
**以便于** 保持工作区整洁

### 验收标准
```
Given 用户正在编辑一个已保存的文件
When 用户点击标签页关闭按钮或按下 Ctrl/Cmd+W
Then 标签页关闭
And 无提示对话框
```

```
Given 用户正在编辑一个未保存的文件
When 用户尝试关闭标签页
Then 弹出对话框"是否保存对 [文件名] 的更改？"
And 提供"保存"、"不保存"、"取消"三个选项
And 选择"保存"后保存并关闭
And 选择"不保存"后直接关闭
And 选择"取消"后不关闭
```

```
Given 只有一个标签页且用户尝试关闭
When 用户执行关闭操作
Then 标签页关闭但编辑器窗口保持打开
And 显示空白编辑区，提示"按 Ctrl/Cmd+O 打开文件"
```

```
Given 最后一个标签页关闭且窗口无其他内容
When 用户关闭窗口
Then 编辑器正常退出
```

### 技术备注
- 关闭多个标签页时，逐个检查未保存状态
- 批量关闭（如"关闭其他标签页"）时，只提示一次"保存所有"

## US-FILE-008: 最近文件列表

### 故事描述
**作为** 频繁切换项目的用户  
**我想要** 快速访问最近打开的文件  
**以便于** 减少查找文件的时间

### 验收标准
```
Given 用户已打开过多个文件
When 用户点击菜单"文件 > 最近打开"
Then 显示最近打开的 10 个文件列表
And 按打开时间倒序排列
And 点击任意文件可快速打开
```

```
Given 某个最近文件已被移动或删除
When 用户尝试从列表中打开
Then 提示"文件不存在，是否从列表中移除？"
And 用户确认后可从列表移除
```

```
Given 用户关闭了某个文件
When 文件关闭
Then 该文件仍保留在最近列表中
And 下次打开时优先显示
```

### 技术备注
- 最近文件列表持久化存储，重启后保留
- 列表上限 20 个文件，超出时移除最旧的
- 敏感路径文件（如系统目录）可配置不记录
