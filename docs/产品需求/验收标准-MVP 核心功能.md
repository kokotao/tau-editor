## AC-001: 文件新建功能

### 验收场景

#### 场景 1: 通过菜单新建文件
```
Given 编辑器已启动且无打开的文件
When 用户点击菜单栏"文件 > 新建"
Then 创建一个新标签页
And 标签页标题显示为"未命名 -1"
And 编辑区为空且可输入
And 光标位于编辑区第一行第一列
```

#### 场景 2: 通过快捷键新建文件
```
Given 编辑器已启动
When 用户按下 Ctrl+N (Windows/Linux) 或 Cmd+N (macOS)
Then 创建一个新标签页
And 新标签页获得焦点
```

#### 场景 3: 连续新建多个文件
```
Given 已存在"未命名 -1"标签页
When 用户再次执行新建操作
Then 创建"未命名 -2"标签页
And 序号依次递增
```

#### 场景 4: 未保存文件关闭提示
```
Given 存在未保存的新建文件
When 用户尝试关闭该标签页
Then 弹出确认对话框
And 对话框显示"是否保存对未命名 -X 的更改？"
And 提供"保存"、"不保存"、"取消"三个按钮
```

### 通过标准
- [ ] 菜单新建功能正常
- [ ] 快捷键新建功能正常
- [ ] 文件命名规则正确
- [ ] 关闭提示正常
- [ ] 新文件默认编码为 UTF-8

## AC-002: 文件打开功能

### 验收场景

#### 场景 1: 通过菜单打开文件
```
Given 编辑器已启动
When 用户点击"文件 > 打开"或按下 Ctrl/Cmd+O
Then 弹出系统文件选择对话框
And 用户可选择文件
And 选中后文件内容显示在编辑区
And 标签页显示文件名
```

#### 场景 2: 拖拽文件打开
```
Given 编辑器窗口已打开
When 用户从文件管理器拖拽文件到编辑器窗口并释放
Then 文件在新标签页中打开
And 文件内容正确显示
```

#### 场景 3: 打开大文件（>10MB）
```
Given 用户选择一个大于 10MB 的文件
When 文件开始加载
Then 显示加载进度条
And 加载完成后文件内容完整显示
And 加载时间不超过 3 秒（10MB 文件）
```

#### 场景 4: 打开不支持的格式
```
Given 用户尝试打开二进制文件（如.exe）
When 文件被检测为不支持格式
Then 显示错误提示"此文件格式不支持"
And 文件不打开
And 不产生崩溃或异常
```

#### 场景 5: 打开超大文件（>500MB）
```
Given 用户尝试打开大于 500MB 的文件
When 文件大小被检测
Then 显示警告"文件较大，可能导致性能问题"
And 用户确认后才继续打开
```

### 通过标准
- [ ] 菜单打开功能正常
- [ ] 拖拽打开功能正常
- [ ] 大文件加载有进度提示
- [ ] 不支持格式有友好提示
- [ ] 文件编码自动检测正确
- [ ] 打开 10MB 文件时间 < 3 秒

## AC-003: 文件保存功能

### 验收场景

#### 场景 1: 新文件保存
```
Given 用户正在编辑一个未保存的新文件
When 用户点击"文件 > 保存"或按下 Ctrl/Cmd+S
Then 弹出系统保存对话框
And 用户输入文件名并选择位置
And 确认后文件保存到指定位置
And 标签页标题更新为实际文件名
And 状态栏显示"已保存"提示
```

#### 场景 2: 已有文件保存
```
Given 用户正在编辑一个已保存过的文件
When 用户执行保存操作
Then 文件直接保存到原位置（无对话框）
And 覆盖原文件内容
And 标签页的"未保存"标识消失
```

#### 场景 3: 保存失败处理
```
Given 保存目标位置无权限或磁盘已满
When 用户执行保存操作
Then 显示错误对话框
And 错误信息说明失败原因
And 提供"重试"、"另存为"、"取消"选项
And 文件保持未保存状态
```

#### 场景 4: 文件外部修改检测
```
Given 文件在编辑器外被其他程序修改
When 用户尝试保存
Then 提示"文件在外部被修改，是否覆盖？"
And 提供"覆盖"、"重新加载"、"取消"选项
```

### 通过标准
- [ ] 新文件保存弹出对话框
- [ ] 已有文件保存直接覆盖
- [ ] 保存失败有明确错误提示
- [ ] 外部修改检测正常
- [ ] 保存操作为原子操作（临时文件 + 替换）

## AC-004: 撤销与重做功能

### 验收场景

#### 场景 1: 撤销操作
```
Given 用户在编辑区输入了文本
When 用户按下 Ctrl/Cmd+Z
Then 最后一次输入被撤销
And 文本恢复到之前的状态
```

#### 场景 2: 连续撤销
```
Given 用户进行了多次编辑操作
When 用户连续按下 Ctrl/Cmd+Z
Then 操作按逆序依次撤销
And 可撤销到文件打开时的状态
```

#### 场景 3: 重做操作
```
Given 用户执行了撤销操作
When 用户按下 Ctrl/Cmd+Shift+Z 或 Ctrl/Cmd+Y
Then 撤销的操作被重做
And 文本恢复到撤销前的状态
```

#### 场景 4: 撤销历史保持
```
Given 用户保存了文件
When 用户继续编辑并撤销
Then 撤销历史不清空
And 可撤销到保存前的状态
```

### 通过标准
- [ ] 撤销功能正常
- [ ] 重做功能正常
- [ ] 撤销历史无限制（受内存限制）
- [ ] 保存后撤销历史保持
- [ ] 撤销/重做快捷键符合平台规范

## AC-005: 查找与替换功能

### 验收场景

#### 场景 1: 打开查找框
```
Given 用户正在编辑文件
When 用户按下 Ctrl/Cmd+F
Then 在编辑器顶部或底部显示查找框
And 查找框获得焦点
And 光标位于查找输入框内
```

#### 场景 2: 查找文本
```
Given 查找框已打开
When 用户输入查找内容
Then 文件中匹配的内容被高亮显示
And 显示匹配结果数量（如"1/5"）
And 支持"上一个"、"下一个"导航
```

#### 场景 3: 区分大小写查找
```
Given 查找框已打开
When 用户勾选"区分大小写"选项
Then 查找时严格区分大小写
And "Hello"和"hello"被视为不同
```

#### 场景 4: 正则表达式查找
```
Given 查找框已打开
When 用户勾选"正则表达式"选项并输入正则模式
Then 按正则表达式匹配内容
And 匹配结果正确高亮
```

#### 场景 5: 替换功能
```
Given 用户按下 Ctrl/Cmd+H 打开替换框
When 用户输入查找内容和替换内容
Then 提供"替换"和"全部替换"按钮
And 点击"替换"替换当前匹配项并跳到下一个
And 点击"全部替换"替换所有匹配项
And 显示替换数量提示
```

### 通过标准
- [ ] 查找框正常打开/关闭
- [ ] 查找内容高亮正确
- [ ] 区分大小写功能正常
- [ ] 正则表达式功能正常
- [ ] 替换功能正常
- [ ] 搜索 10000 行文件时间 < 1 秒

## AC-011: 自动保存功能

### 验收场景

#### 场景 1: 开启自动保存
```
Given 用户在设置中开启自动保存
When 设置保存后
Then 自动保存功能激活
And 状态栏显示自动保存状态
```

#### 场景 2: 自动保存触发
```
Given 自动保存已开启且间隔为 5 分钟
When 用户编辑文件且 5 分钟内未手动保存
Then 编辑器自动保存当前内容
And 状态栏显示"自动保存于 HH:MM"

```

#### 场景 3: 配置保存间隔
```
Given 用户在设置中配置自动保存
When 用户选择保存间隔
Then 可选值：1 分钟、5 分钟、10 分钟、30 分钟、关闭
And 设置立即生效
```

#### 场景 4: 崩溃恢复
```
Given 自动保存已开启且用户编辑了内容
When 编辑器意外崩溃
Then 重新启动编辑器时
And 提示"发现未保存的会话，是否恢复？"
And 用户确认后可恢复到最后一次自动保存状态
```

### 通过标准
- [ ] 自动保存开关正常
- [ ] 保存间隔可配置
- [ ] 自动保存不中断编辑
- [ ] 崩溃恢复功能正常
- [ ] 状态栏显示正确
