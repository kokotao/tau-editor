## 2. 手工验收步骤

## A. 关闭按钮（叉叉）可用性

1. 新建文件并输入任意文本（制造未保存状态）。
2. 点击窗口关闭按钮。
3. 验证出现“未保存更改”确认弹窗。
4. 点击“取消”，验证应用仍保持打开。
5. 再次点击关闭按钮，点击“退出”，验证应用可正常关闭。

## B. Markdown 实时预览 + Mermaid

1. 新建文件。
2. 状态栏语言切换到 `Markdown`。
3. 输入：
   - `# 标题测试`
   - Mermaid 代码块：
     ````
     ```mermaid
     graph TD
       A --> B
     ```
     ````
4. 验证右侧预览实时更新，能看到标题和流程图。
5. 点击工具栏 Markdown 视图按钮多次，验证 `仅编辑 -> 分栏 -> 仅预览` 循环切换。

## C. 启动和打开提示精简

1. 启动应用。
2. 验证不出现“编辑器已就绪”提示。
3. 打开文件/文件夹。
4. 验证不出现“文件已打开/工作区已打开”等成功通知。
5. 故意打开不存在路径（或触发失败场景），验证仍有错误提示。

## D. 左侧面板折叠/展开与拖拽

1. 点击工具栏资源管理器按钮，验证侧栏折叠。
2. 点击左侧浮动展开按钮，验证侧栏恢复。
3. 拖拽侧栏边界分隔条，验证宽度变化。
4. 使用 `Ctrl/Cmd + B`，验证可切换侧栏显隐。
5. 关闭并重启应用，验证侧栏显隐状态与宽度记忆。

## E. 资源管理器目录结构优化

1. 打开含多级目录的项目。
2. 验证目录箭头可准确折叠/展开，不误触。
3. 验证文件夹图标在展开/折叠时状态正确。
4. 键盘导航验证：
   - `ArrowRight` 展开
   - `ArrowLeft` 折叠/聚焦父级
   - `Enter` 打开文件
5. 折叠左侧面板后再展开，验证原目录展开状态仍保持。
